---
title: Quick Start
description: Fast path from empty host to one disposable service.
---

This is the shortest useful path: install `yeet`, prepare Tailscale, install
`catch`, run a disposable container, and remove it.

## TLDR

You need:

- A Linux host with systemd and SSH access.
- Tailscale installed on your workstation and connected to the same tailnet.
- Tailscale admin access for first setup.
- A Tailscale OAuth client secret for catch.
- Docker on the host for the disposable container test.

Skip VMs for the first run. Containers prove the basic path with less host
drama.

## 1. Prepare Tailscale

Do this before `yeet init`.

Catch joins your tailnet as a tagged device, usually `tag:catch`. Your setup
user also needs the `yeetrun.com/app/yeet` permissions `read`, `manage`, and
`ssh` on TCP `41548`.

The short version:

1. Open [Tailscale Setup](/docs/concepts/tailscale#first-time-host-setup).
2. Create the tag policy and OAuth credential there.
3. Keep the `tskey-client-...` OAuth client secret ready.
4. Make sure your workstation is connected to the same tailnet.

## 2. Install yeet

Run this on your workstation:

```bash
curl -fsSL https://yeetrun.com/install.sh | sh
```

Check that the CLI is available:

```bash
yeet --help
```

For install options, nightly builds, and upgrades, use the
[Installation guide](/docs/getting-started/installation).

## 3. Install catch on one host

Run this from your workstation. The target is the SSH machine host:

```bash
yeet init root@<machine-host>
```

During first setup, paste the Tailscale OAuth client secret from step 1.
If Docker is missing on a Debian/Ubuntu-style host, interactive init asks before
installing it.

Use `root@<machine-host>` for SSH setup. After setup, normal commands target
the catch hostname on your tailnet.

## 4. Confirm yeet can reach catch

If setup saved a default host, these should work:

```bash
yeet version
yeet status
```

If you have more than one catch host, be explicit:

```bash
yeet --host=<catch-host> status
```

## 5. Create a service workspace

Run services from a directory you plan to keep. A successful deploy writes
`yeet.toml` there.

```bash
mkdir -p ~/yeet-services
cd ~/yeet-services
```

This is boring local state. Good. Hidden state is how simple tools become
surprises.

## 6. Run a disposable service

Start with nginx. It is small, familiar, and easy to delete.

```bash
yeet run -p 18080:80 hello nginx:alpine
yeet status hello
yeet logs hello
```

Check the published port from the host:

```bash
yeet ssh -- curl -fsS http://127.0.0.1:18080/ >/dev/null
```

## 7. Clean it up

```bash
yeet rm --clean hello
```

`--clean` deletes service data and removes the disposable `yeet.toml` entry.
Read the prompt. Cleanup is not the place for autopilot.
