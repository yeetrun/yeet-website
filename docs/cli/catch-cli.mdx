Catch runs on remote Linux hosts and receives requests from yeet. For normal
operation, install and update catch with `yeet init`.

## Usage

```bash
catch [flags] [install|version|is-catch]
```

## Commands

- `install`: install catch as a system service on the host.
- `version`: print the catch build version.
- `is-catch`: print `yes` for detection.

## Common flags

- `--data-dir`: base directory for catch state. Installed catch uses the data
  directory selected during install. Manual runs default to `data` under the
  current working directory.
- `--tsnet-host`: catch tsnet hostname. Default: `catch`.
- `--tsnet-port`: catch tsnet port.

See [Data Layout](/docs/concepts/data-layout) for host storage and
[Tailscale](/docs/concepts/tailscale) for tsnet setup.

## Docker during install

During `catch install`, catch requires Docker for container payloads. If Docker
is missing, interactive installs can prompt to install it.

For normal installs, prefer:

```bash
yeet init --install-docker root@<machine-host>
```

For manual non-interactive catch installs on Debian/Ubuntu-style hosts, set:

```bash
CATCH_INSTALL_DOCKER=1
```

## When to run catch directly

Most users should not need direct catch commands. Use direct catch invocation
only when you are installing manually, checking the catch version on the host,
or debugging catch startup.

Use [Installation](/docs/getting-started/installation) for the supported
bootstrap path and [yeet Commands](/docs/cli/yeet-cli) for normal operations.
