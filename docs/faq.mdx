## Is this intended for general use?

Yeet is intended for homelab-style use by people comfortable operating Linux
hosts over SSH. It is not a general-purpose platform or a multi-tenant
orchestrator: hosts are expected to run Linux/systemd, RPC uses Tailscale/tsnet,
and catch currently manages services as root.

## Does yeet require Tailscale?

Yes. Yeet communicates with catch over Tailscale RPC. See
[Tailscale](/docs/concepts/tailscale).

## Do services run as root?

Yes. This is acceptable for a single-operator homelab, but not recommended for
shared or multi-tenant environments.

## Can I use docker compose?

Yes. Provide a compose file to `yeet run`.

## How do I update a service?

Re-run `yeet run` with the updated payload. For compose services, `yeet run`
does not pull images by default; use `yeet run --pull <svc> ./compose.yml` or
`yeet docker update <svc...>` (pulls images and recreates containers) when you
want fresh images for selected services. Use `yeet docker update --outdated` to
update all compose services with available image updates.
